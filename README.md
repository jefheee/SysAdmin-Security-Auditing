#  Auditoria de Sistemas e Hardening de Segurança

Este projeto acadêmico aprofundado foca na arquitetura de sistemas operacionais e na implementação de políticas de segurança rigorosas para proteção de ambientes corporativos contra vulnerabilidades externas e internas.

##  Arquitetura de Segurança
O trabalho consistiu em uma análise técnica detalhada sobre como estruturar a defesa de um sistema, abordando desde a camada de hardware até as políticas de governança de dados:
* **Controle de Acesso e Permissões:** Estudo e implementação de privilégios de usuário em ambientes Linux e Windows, utilizando o princípio do menor privilégio para mitigar riscos de escalação de permissões.
* **Hardening de Sistemas:** Aplicação de diretrizes para "endurecer" o sistema operacional, desativando serviços desnecessários e portas vulneráveis para reduzir a superfície de ataque.
* **Gestão de Vulnerabilidades:** Análise crítica de falhas comuns em infraestruturas locais e definição de protocolos de remediação e correção de imagem.

##  Funcionalidades e Estudos Realizados
* **Políticas de Proteção de Dados:** Definição de diretrizes para garantir a tríade da segurança da informação: Integridade, Disponibilidade e Confidencialidade.
* **Auditoria Computacional:** Procedimentos técnicos para monitoramento de logs de sistema e detecção de comportamentos anômalos no ambiente.
* **Arquitetura de Hardware Segura:** Estudo sobre como os componentes físicos interagem com a camada de segurança do kernel.

##  Tecnologias e Conceitos
* **Sistemas Operacionais:** Administração avançada de distribuições Linux e versões Windows Server.
* **Cibersegurança:** Hardening, ACLs (Access Control Lists) e Auditoria de Sistemas.
